# 🍱 점심메뉴 추천

점심 메뉴 고르기가 힘든 사람을 위한 PWA입니다. 서버 없이 정적 파일(`index.html`)만으로 동작합니다.

## 기능

1. **선호 음식 유형 선택** — 한식/중식/일식/양식/분식/아시안/패스트푸드/샐러드 중 좋아하는 유형을 골라두면 기기(localStorage)에 저장됩니다.
2. **GPS 기반 추천** — "내 위치 기반 추천 받기"를 누르면 브라우저 위치 권한으로 현재 위치를 확인하고, 선호 유형에 가중치(5배)를 줘서 메뉴를 추천합니다.
3. **먹은 메뉴 기록** — "오늘 이거 먹음"으로 기록하면 최근 3일간 같은 메뉴가 추천에서 제외됩니다.
4. **리뷰 확인** — 추천 메뉴 기준으로 네이버 지도/카카오맵 검색이 바로 열려 주변 맛집 리뷰를 확인할 수 있습니다.
5. **주변 음식점 실시간 검색 (선택)** — [카카오 디벨로퍼스](https://developers.kakao.com/)에서 발급한 REST API 키를 입력하면, 추천 메뉴 기준 반경 1km 음식점을 거리순으로 보여주고 각 가게의 카카오맵 리뷰 페이지로 연결합니다. 키는 localStorage에만 저장됩니다.

## 네이버 리뷰 취합에 대해

네이버 검색(지역) API와 플레이스 리뷰는 **브라우저에서 직접 호출할 수 없습니다** (CORS 차단 + API 키가 클라이언트에 노출되는 문제). 그래서 현재는:

- 네이버 지도 **검색 딥링크**로 연결해 리뷰를 바로 보게 하고,
- 실데이터가 필요하면 CORS를 허용하는 **카카오 로컬 API**(선택)를 사용합니다.

네이버 리뷰 점수까지 자동 취합하려면 작은 백엔드(프록시)가 필요합니다. 예: Cloudflare Workers/Vercel Functions에 네이버 지역검색 API 프록시를 두고 `평점·리뷰 수`를 받아 추천 점수에 반영하는 구조. 필요해지면 그때 추가하면 됩니다.

## 실행

정적 호스팅(GitHub Pages 등)에 올리거나 로컬에서:

```bash
python3 -m http.server 8000
# http://localhost:8000
```

> 위치 권한(Geolocation)은 `https` 또는 `localhost`에서만 동작합니다.
