# 🍱 점심메뉴 추천

점심 메뉴 고르기가 힘든 사람을 위한 PWA입니다. 서버도, API 키도 없이 정적 파일(`index.html`)만으로 동작합니다.

## 기능

1. **선호 음식 유형 선택** — 한식/중식/일식/양식/분식/아시안/패스트푸드/샐러드 중 좋아하는 유형을 골라두면 기기(localStorage)에 저장됩니다.
2. **메뉴 추천** — "점심 메뉴 추천 받기"를 누르면 선호 유형에 가중치(5배)를 줘서 메뉴 1개와 차선책 2개를 추천합니다.
3. **날씨 / 기분 반영** — 날씨(추움/비/더움/보통)와 기분(얼큰한/든든한/가벼운/해장)을 고르면 **그 분위기에 맞는 메뉴 중에서만** 추천합니다(예: 얼큰한 → 짬뽕·마라탕·부대찌개 등). 둘 다 골랐을 때 겹치는 메뉴가 없으면 두 조건의 합집합으로 완화합니다. "현재 날씨 자동" 버튼은 입력한 지역(동 이름)으로 [Open-Meteo](https://open-meteo.com/)(무료·키 불필요)에서 현재 날씨를 가져와 자동 선택합니다.
4. **지역 + 메뉴 지도 검색** — 지역(동/역 이름)과 메뉴를 입력하면 네이버 지도/카카오맵에서 "지역 + 메뉴 맛집"으로 바로 검색합니다. 추천 메뉴와 직접 입력한 메뉴 모두 검색할 수 있고, 지역은 localStorage에 저장됩니다.
5. **먹은 메뉴 기록** — "오늘 이거 먹음"으로 기록하면 최근 3일간 같은 메뉴가 추천에서 제외됩니다.

## 왜 API 키가 없나요?

지도/장소 검색을 **브라우저에서 직접 API로 호출**하려면 키 발급·플랫폼 등록·서비스 활성화(카카오의 경우 비즈니스 앱 전환)가 필요하고, 키가 클라이언트에 노출되는 문제도 있습니다. 그래서 이 앱은 **API 호출 대신 지도 검색 딥링크**(`map.naver.com`, `map.kakao.com`)로 연결합니다 — 키·권한·CORS 문제 없이 누구나 바로 쓸 수 있습니다.

> 앱 화면 안에 "주변 음식점 거리순 리스트"까지 넣으려면 작은 백엔드(프록시)로 지도 장소 API를 호출하는 구조가 필요합니다. 필요해지면 그때 추가하면 됩니다.

## 실행

정적 호스팅(GitHub Pages 등)에 올리거나 로컬에서:

```bash
python3 -m http.server 8000
# http://localhost:8000
```
